compilerOptions

Property Meaning
alwaysStrict Parse in strict mode, add "use strict" to each file
declaration Allows to automatically generate *.d.ts files
downlevelIteration Provide full support for iterables in for..of, spread and destructuring when targeting ES5 or ES3.
target Target ES version eg. es2015, es2017 etc.
noImplicitAny Raise error on expressions and declarations with an implied any type.
module Module code generation, eg. ES6, AMD, CommonJS etc. see next chapter for more inforation
moduleResolution Module resolution, eg. ES6, AMD, CommonJS etc. see documentation for details
allowUmdGlobalAccess Allow accessing UMD globals from modules. More details here.
allowUnreachableCode Do not report errors on unreachable code.
baseUrl or paths Base paths for nonrelative module names
typeRoots Locations (base paths) of type definitions
types Names of *.d.ts files with type definitions
experimentalDecorators Allows using decorators -it was experimental feature introduced in ES7
lib List of libraries (JS features) used in code. eg. esnext.Array, es2018.promise etc...
noEmitOnError Don't generate outputs when errors found.
sourceMap Generates corresponding .map file.

include/exclude or files

Specifies which files should be compiled.
